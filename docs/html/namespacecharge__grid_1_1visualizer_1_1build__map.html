Fetches real-world driving geometries for active customer-station assignments.

Iterates over the flow matrix. For every active connection (F_matrix[i][j] > 0), it queries the public OSRM (Open Source Routing Machine) API using the geographic coordinates from df_meta to retrieve exact road pathways.

Function Connections:
  • Called by generate_interactive_map immediately after build_map_data.
  • Appends the fetched GeoJSON coordinates directly into the ‘map_data['routes’]` list.
Parameters
map_dataThe visualization dictionary generated by build_map_data.
F_matrixA 2D NumPy array representing the fulfilled demand flows.
df_metaA pandas DataFrame containing node coordinates.
NTotal number of nodes.
LThe distance matrix.
sessionA requests.Session() object for connection pooling.
Returns
The updated map_data dictionary containing the real-world route geometries.

Constructs the structured data dictionaries required for front-end visualization.

Iterates through the network matrices to build a comprehensive JSON-serializable dictionary. It aggregates data for active stations (calculating their specific profit and dissatisfaction), unselected stations, customer nodes (tracking met vs. unmet demand), direct routes, and spatial heat map coordinates.

Function Connections:
  • Called by generate_interactive_map.
  • Relies on the F_matrix generated by the selected routing model to link customers and stations.
Parameters
best_xA list of integers (1s and 0s) representing the active stations.
F_matrixA 2D NumPy array representing the fulfilled demand flows.
df_metaA pandas DataFrame containing the geographic coordinates and names of nodes.
N,D,L,C,P,RCore network parameters (Nodes, Demand, Distances, Costs, Profit, Recurring Costs).
configThe experiment configuration dictionary.
Returns
A dictionary containing keys: 'stations', 'unselected_stations', 'customers', 'routes', and 'heat_data'.

Prompts the user via CLI to select a saved optimization result.

Scans the predefined OUTPUT_DIR for JSON files starting with 'solution_'. Displays an indexed list in the console and accepts integer input to select the desired file for visualization.

Function Connections:
  • Called in the main execution block to determine which solution to load and map.
Returns
A pathlib.Path object pointing to the chosen JSON file, or None if the directory is empty or the input is invalid.

Orchestrates the creation and export of the interactive HTML map.

Recalculates the flow matrix F using the dynamically loaded routing model. It computes the overall economic (E) and operational (O) metrics for the UI overlay. It then builds the spatial data structures, retrieves OSRM routes, constructs a base folium.Map, and injects custom CSS, JavaScript, and JSON data directly into the HTML DOM before saving the file to the cache directory.

Function Connections:
  • Calls model.route() or model.solve_flow() to resolve the flow matrix for the saved layout.
  • Calls E and O (from utils.py) to recalculate total metrics.
  • Calls build_map_data and append_osrm_routes to generate the payload.
  • Reads external assets (map_style.css, map_script.js) and binds them to the HTML.
Parameters
best_xThe optimal station layout array from the loaded JSON file.
df_metaPandas DataFrame of geographic metadata.
N,B,C,P,R,Z,D,LCore network parameters.
configThe experiment configuration dictionary.
modelThe instantiated routing model object (e.g., CustomerRouting).
shuffle_seedInteger seed used if the model relies on randomized order evaluation.
