Solves the optimal EV station placement problem using a Mixed-Integer Linear Programming (MILP) solver.

This function provides an exact mathematical baseline for the routing optimization using Google OR-Tools (SCIP solver). It defines binary decision variables for station placement ($x_j$) and integer variables for demand flow ($F_{i,j}$).

The solver maximizes the objective function: $$\text{Maximize } \lambda(E) - O$$ Where $E$ is total profit (revenue - costs) and $O$ is the combined operational penalty (unmet demand + distance traveled).

It enforces the following strict mathematical constraints:

  • Demand Limit: $\sum_j F_{i,j} \le D_i$
  • Distance Tolerance: $F_{i,j} = 0 \text{ if } L_{i,j} > Z_i$
  • Station Activation: $F_{i,j} \le D_i \cdot x_j$
  • Station Capacity: $\sum_i F_{i,j} \le B \cdot x_j$
Function Connections:
  • Relies on read_input (from charge_grid.utils) to parse the network matrices.
  • Serves as a deterministic, exact-solution counterpart to the heuristic Genetic Algorithm models used elsewhere in the project.
Parameters
input_fileA string or Path pointing to the network dataset text file.
alphaA float representing the penalty multiplier for unmet customer demand.
betaA float representing the penalty multiplier for customer travel distance.
lambda_weightA float used to scale the economic profit component of the objective.
time_limit_msAn optional integer defining the maximum milliseconds the solver is permitted to run before forcing termination.
Returns
A tuple (best_x, val_fitness), where best_x is a list of integers (1s and 0s) representing the optimal layout, and val_fitness is the maximum objective value found. Returns (None, None) if the solver cannot find a feasible solution.
