charge_grid.utils.adaptive_mutation ( data_tuple)

A factory function that generates a demand-aware adaptive mutation operator.

This function extracts the customer demand vector D from the provided data tuple to create a custom mutation closure for PyGAD.

The generated mutation operator iterates through each chromosome in the offspring array. For each chromosome, it performs a coin toss (50% probability):

  • Add a station: It identifies all empty nodes (0s). It then calculates a probability distribution based on the localized demand at those empty nodes. A new station is added (flipped to 1), heavily favoring locations with high unfulfilled demand.
  • Remove a station: It identifies all currently active stations (1s) and drops one (flipped to 0) uniformly at random.
Function Connections:
  • Extracts D (demand vector) from the globally loaded data_tuple.
  • Assigned to the mutation_type key in the Experiment configuration dictionary.
  • The returned inner function _adaptive_mutation is called automatically by the PyGAD engine to mutate offspring chromosomes before the next generation.
Parameters
data_tupleA tuple containing the extracted network parameters, specifically utilizing the initial demand vector at index 7 (D).
Returns
A callable function _adaptive_mutation(offspring, ga_instance) that complies with PyGAD's custom mutation signature and returns the modified offspring array.

charge_grid.utils.custom_intersection_crossover ( data_tuple)

A factory function that generates a spatially-aware custom crossover operator.

This function acts as a closure, extracting the distance matrix L from the provided data tuple to be used during the crossover phase. It returns a custom crossover function formatted for PyGAD.

The generated crossover algorithm uses a dynamic clustering approach to preserve geographic structures. For each pair of parents, it randomly selects up to K (10) medoid nodes and groups all other nodes into clusters based on their proximity in the distance matrix L. Offspring then inherit entire geographical clusters of genes from either the first or second parent with a 50% probability, preventing the fragmentation of highly optimized local station networks.

Function Connections:
  • Extracts L (distance matrix) from the globally loaded data_tuple.
  • Assigned to the crossover_type key in the Experiment configuration dictionary.
  • The returned inner function _custom_intersection_crossover is called automatically by the PyGAD engine during the evolution phase to breed new chromosomes.
Parameters
data_tupleA tuple containing the extracted network parameters, specifically utilizing the distance matrix at index 4 (L).
Returns
A callable function _custom_intersection_crossover(parents, offspring_size, ga_instance) that complies with PyGAD's custom crossover signature and returns a NumPy array of generated offspring.

Computes the economic metrics (profit, revenue, and cost) for a given network layout.

Calculates the total financial performance of the current EV station configuration. Revenue is derived from the total fulfilled demand (the flow matrix F routed to active stations x) multiplied by the unit profit P. Cost is computed as the sum of base costs C and recurring costs R for all actively deployed stations. The total profit is the difference between this revenue and cost.

Function Connections:
  • Imported and utilized by routing models (e.g., AlternatingRouting.fitness and AlternatingRouting.get_details) to evaluate the financial fitness of a chromosome.
Parameters
xA NumPy array or list of integers (1s and 0s) indicating active station locations.
FA 2D NumPy array representing the flow of demand from customers to stations.
CA scalar or NumPy array representing the base cost of building the stations.
PA float representing the revenue or profit gained per unit of demand served.
RA scalar or NumPy array representing additional/recurring costs per station.
Returns
A tuple of three floats: (total_profit, total_revenue, total_cost).

charge_grid.utils.noise_injected_adaptive_mutation ( data_tuple)

A factory function generating a state-aware mutation operator with random noise injection.

This function builds upon the stagnation_aware_adaptive_mutation logic by introducing a baseline probability of completely random bit-flipping.

For each chromosome in the offspring array:

  1. Noise Injection: There is a 10% chance to bypass all adaptive logic entirely and simply flip one random gene (station state). This guarantees a baseline level of genetic diversity to prevent premature convergence.
  2. Stagnation Recovery: If PyGAD detects stagnation (>50 generations without improvement), it intelligently drops 20% of the active stations, heavily targeting geographically redundant clusters.
  3. Dynamic Normal Operation: If neither of the above apply, it scales the probability of adding vs. dropping a station based on the current network utilization (Demand / Capacity). When adding, it favors high-demand nodes; when dropping, it targets redundant nodes.
Function Connections:
  • Extracts D (demand), L (distances), and B (capacity) from the globally loaded data_tuple.
  • Assigned to the mutation_type key in the Experiment configuration dictionary.
  • Interacts directly with PyGAD's internal state tracking (ga_instance.stagnation_counter).
Parameters
data_tupleA tuple containing the extracted network parameters.
Returns
A callable function _noise_injected_adaptive_mutation(offspring, ga_instance) that applies the hybrid noise/adaptive mutations and returns the modified offspring array.

Computes the operational penalty metrics for unmet demand and travel distance.

Evaluates the operational inefficiency and customer dissatisfaction of the current routing assignment. It calculates the total unfulfilled demand by subtracting the total served flow from the initial demand vector D. It also calculates the total distance traveled by multiplying the flow matrix F by the distance matrix L. These raw values are scaled by their respective penalty weights (alpha and beta) to produce a final, combined penalty score.

Function Connections:
  • Imported and utilized by routing models (e.g., AlternatingRouting.fitness and AlternatingRouting.get_details) to apply negative pressure against inefficient layouts during the genetic algorithm's evaluation phase.
Parameters
FA 2D NumPy array representing the flow of demand from customers to stations.
DA 1D NumPy array representing the total initial demand of each customer node.
LA 2D NumPy array representing the distance matrix between customers and stations.
alphaA float acting as the penalty multiplier for each unit of unfulfilled demand.
betaA float acting as the penalty multiplier for each unit of distance traveled.
Returns
A tuple of three floats: (total_combined_penalty, unmet_demand_penalty, distance_penalty).

charge_grid.utils.read_input ( input_path)

I/O Functions.

Reads and parses the EV network parameters from a specified text file.

Opens the given file path and extracts scalar parameters (number of nodes, battery capacity, cost, profit) from the first line. It then parses the subsequent lines into NumPy arrays representing the distance matrix, cost vectors, distance tolerances, and nodal demands. Raises a FileNotFoundError if the file does not exist.

Function Connections:
  • Acts as the primary data source for the entire project.
  • Called globally in main.py to generate the data_tuple.
  • The returned tuple is subsequently unpacked by Experiment.__init__ and passed into routing models (like AlternatingRouting) and the evaluation functions (E and O).
Parameters
input_pathA pathlib.Path object pointing to the input dataset file.
Exceptions
FileNotFoundErrorRaised if the provided `input_path` does not exist on disk.
Returns
A tuple containing (N, B, C, P, L, R, Z, D), where N and B are integers, C and P are floats, and L, R, Z, D are structured NumPy arrays.

charge_grid.utils.stagnation_aware_adaptive_mutation ( data_tuple)

A factory function generating an advanced, state-aware adaptive mutation operator.

This function extracts demand (D), distances (L), and station capacity (B) from the data tuple to construct a highly dynamic PyGAD mutation operator. The operator adjusts its behavior based on the current state of the genetic algorithm and the specific characteristics of each chromosome:

  1. Stagnation Recovery: If PyGAD detects no fitness improvement for over 50 generations, the operator forces a major structural shift to escape local optima. It identifies active stations that are geographically too close to one another (redundant) and simultaneously drops 20% of them.
  2. Dynamic Normal Operation: If not stagnant, the operator calculates the network's current utilization (Total Demand / Total Network Capacity).
    • High utilization increases the probability of adding a new station (favoring high-demand nodes).
    • Low utilization increases the probability of dropping a station (specifically targeting redundant stations clustered too closely together).
Function Connections:
  • Extracts D, L, and B from the globally loaded data_tuple.
  • Assigned to the mutation_type key in the Experiment configuration dictionary.
  • Interacts directly with PyGAD's internal state tracking (ga_instance.stagnation_counter) to detect local optima.
Parameters
data_tupleA tuple containing the extracted network parameters.
Returns
A callable function _stagnation_aware_adaptive_mutation(offspring, ga_instance) that applies state-dependent mutations and returns the modified offspring array.
