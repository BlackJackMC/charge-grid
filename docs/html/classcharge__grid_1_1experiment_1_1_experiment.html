Detailed Description

A controller class for managing the genetic algorithm optimization process.

This class acts as a dedicated wrapper around the PyGAD library. It decouples the core genetic algorithm mechanics from the specific domain logic (the routing models). It is responsible for setting up the GA engine, managing the state and history of the population across generations, evaluating chromosomes using the injected model, and handling the final data export.

Class Connections:
  • Integrates with pygad.GA to drive the evolutionary loop.
  • Dynamically builds and interacts with a specific domain model (e.g., AlternatingRouting, CustomerRouting) provided via the configuration to compute fitness scores.
  • Relies on external I/O utilities (like save_optimization_results from utils.py) to persist the optimization outcomes to disk.

Initializes the Experiment instance and prepares the routing model.

Unpacks the network parameters from the provided data tuple and stores the experiment's metadata and configuration. It dynamically instantiates the specific routing model defined in the configuration dictionary and initializes an empty list to track the generation history during execution.

Function Connections:
  • Extracts the class reference from ‘config['model_builder’](e.g.,AlternatingRouting`) and invokes its constructor, passing the unpacked network parameters to build the core evaluation engine.
Parameters
dataA tuple containing the extracted network parameters (N, B, C, P, L, R, Z, D).
experiment_nameA string representing the name of the current experiment.
input_pathA Path object pointing to the input dataset file.
output_folderA Path object designating the directory for saving results.
configA dictionary containing the genetic algorithm hyperparameters and the model builder class reference.

A cached helper function to store and retrieve fitness calculations.

This method uses Python's @cache to store the fitness scores of previously evaluated station layouts. If the provided tuple exists in the cache, it returns the stored score; otherwise, it calculates the fitness for the solution and caches the result for future lookups.

Parameters

The fitness score as a float.

A callback executed at the end of each generation to record metrics.

Retrieves the best solution and population fitness statistics from the current generation. It calculates detailed routing metrics (revenue, cost, unmet demand, distance) for the best solution, appends these details to the generation history list, and prints a summary to the console.

◆ run()

charge_grid.experiment.Experiment.run ( self)

Initializes and executes the genetic algorithm optimization process.

This method configures the PyGAD engine using the parameters stored in self.config and starts the evolutionary loop. It tracks the total execution time, retrieves the highest-scoring chromosome after the loop finishes, and saves results to a JSON file.

Function Connections:
  • Uses pygad.GA() to instantiate the external genetic algorithm engine.
  • Links self.fitness_handler to PyGAD to score chromosomes during the run.
  • Links self.log_handler to PyGAD to record metrics at the end of each generation.
  • Calls ga_instance.run() to trigger the internal PyGAD optimization loop.
  • Calls save_optimization_results (from utils.py) to persist the final solution, configuration, and generation history to disk.
Returns
A tuple (best_x, best_fitness), where best_x is a list of integers representing the optimal station layout, and best_fitness is the corresponding float score.
