\doxysubsection{Detailed Description}
A controller class for managing the genetic algorithm optimization process. 

This class acts as a dedicated wrapper around the Py\+GAD library. It decouples the core genetic algorithm mechanics from the specific domain logic (the routing models). It is responsible for setting up the GA engine, managing the state and history of the population across generations, evaluating chromosomes using the injected model, and handling the final data export.

\begin{DoxyParagraph}{Class Connections\+:}

\begin{DoxyItemize}
\item Integrates with {\ttfamily pygad.\+GA} to drive the evolutionary loop.
\item Dynamically builds and interacts with a specific domain model (e.\+g., {\ttfamily Alternating\+Routing}, {\ttfamily Customer\+Routing}) provided via the configuration to compute fitness scores.
\item Relies on external I/O utilities (like {\ttfamily save\+\_\+optimization\+\_\+results} from {\ttfamily \doxylink{utils_8py}{utils.\+py}}) to persist the optimization outcomes to disk. 
\end{DoxyItemize}
\end{DoxyParagraph}

{\footnotesize\ttfamily charge\+\_\+grid.\+experiment.\+Experiment.\+run (\begin{DoxyParamCaption}\item[{}]{self }\end{DoxyParamCaption})}



Initializes and executes the genetic algorithm optimization process. 

This method configures the Py\+GAD engine using the parameters stored in {\ttfamily self.\+config} and starts the evolutionary loop. It tracks the total execution time, retrieves the highest-\/scoring chromosome after the loop finishes, and saves results to a JSON file.

\item Links {\ttfamily self.\+fitness\+\_\+handler} to Py\+GAD to score chromosomes during the run.
\item Links {\ttfamily self.\+log\+\_\+handler} to Py\+GAD to record metrics at the end of each generation.

\item Calls {\ttfamily save\+\_\+optimization\+\_\+results} (from {\ttfamily \doxylink{utils_8py}{utils.\+py}}) to persist the final solution, configuration, and generation history to disk.
